Nonogram Puzzle

Your task is to create simple interface to solve logic puzzles known as Nonograms.
Nonogram is a puzzle where you have to paint the fields (or leave the field unpainted) on the board accordingly to the numbers that are shown beside rows and above columns. Numbers beside rows indicate how many unbroken by white spaces lines there are in the row and how long they are. Analogously, numbers above each column indicate number of unbroken lines and the their length for each column.
For example, clue '4 5 1' beside row indicate that there is a sequence of 4 painted black fields, then there is some space made by unpainted fields (it is not specified how long), then sequence of 5 painted fields, then again space and 1 painted block. White space can be present before first painted sequence or last painted field.
Goal of this game is to paint those fields so that every row and column clue is used and row/column painted fields don't contradict with each other. Usually, painted fields create some sort of a painting.

  • Example application
  • Task description:
    • Main window and controls:
      • Main window has a size of 1000x800 and starts centered on the screen
      • Main window is non-resizable and has maximalize button disabled
      • On the top of the window there is a menu bar with 2 options: 'New Game' and 'Create...'
      • Under 'New Game' there are three options: 'Random', 'Choose puzzle...' and 'Load puzzle' (disabled in lab part)
      • Under 'Create...' there is one option: 'Create puzzle'(disabled in lab part)
      • The main window area is composed from:
        • board 10x10 (each tile has size of 30x30)
        • labels beside each row and above each column
    • Board:
      • Start board size : 10x10 tiles
      • Each tile size: 30px x 30px
      • Each tile color: white when empty, black when filled
      • There shouldn't be any space between tiles in the board
      • Board should be centered in the window
      • On the start of the application, there is a random puzzle generated and the board is already painted accordingly to the generated board values
    • Labels:
      • Labels are shown beside each row and each column
      • Number in row indicate how many unbroken painted lines there are in this row and how long they are
      • Analogously, column text indicate how many unbroken painted lines there are in this column and how long they are
      • Numbers in row labels are separated by a little space (to indicate they are separate)
      • Row labels are aligned to left side of the board, to the center of the row's tiles
      • Column labels are aligned to top side of the board, to the center of the column's tiles
    • When clicking on specific field:
      • If the field was empty, it should be filled with black color
      • If the field was filled, it should be emptied with white color
  • Tips:
    • MenuStrip, Items property
    • FormBorderStyle property
    • Event MouseDown
    • TextAlign property
    • TabStop, FlatStyle properties
  • Note: In all doubtful and untold aspects application should behave like an example app.

    • Labels:
      • Drawing labels beside rows and above columns: 1 point
      • Correctly calculating numbers for labels: 1 point
      • Correctly positioning and aligning row/column labels to middle right and middle bottom respectively: 2 points
    • Clicking on the fields change field color: 1 point
